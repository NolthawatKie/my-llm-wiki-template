# gap-agent

## Role

Strategic analyst. Reads the roadmap and the current state of the wiki, identifies what is missing or under-covered, scores each gap, and produces an actionable study queue. Requires judgment about importance and learning sequencing — uses sonnet.

## Inputs

- `focus` — (optional) constrain analysis to a roadmap section, e.g. `"transformers"` or `"rlhf"`. Default: full roadmap.
- `queue_size` — (optional, int) number of items to write into study-queue.md. Default: 10.

## Steps

### Phase 1 — Load Context

1. Read `wiki/gaps/roadmap.md` in full.
2. Read `wiki/gaps/list.md` (existing known gaps).
3. Read `wiki/index.md` to get the full inventory of existing wiki pages.
4. Read `wiki/memory/sessions/latest.md` to understand what was recently studied.

### Phase 2 — Coverage Mapping

For each topic node in the roadmap:
1. Check whether a corresponding concept or entity page exists in the wiki.
2. If a page exists, assess coverage depth:
   - `shallow` — page exists but has < 3 source_files or < 200 words.
   - `partial` — page exists with some coverage but missing key sub-topics.
   - `covered` — page is substantive and cross-linked.
3. Build a coverage map:
   ```
   topic | status | existing_page | notes
   ```

### Phase 3 — Gap Scoring

For each uncovered or shallow topic, assign a score (1–10) across three dimensions:

| dimension | weight | description |
|---|---|---|
| `importance` | 40% | How foundational is this to the AI/LLM Engineer path? |
| `dependency` | 35% | How many other roadmap topics depend on knowing this first? |
| `proximity` | 25% | How close is this to topics already well-covered in the wiki? (learn adjacent first) |

Final score = weighted average. Round to one decimal place.

### Phase 4 — Rewrite gaps/list.md

Overwrite `wiki/gaps/list.md` with the full updated gap list:

```markdown
---
title: Knowledge Gaps
type: gap-list
updated: <today>
---

# Knowledge Gaps

| topic | status | score | roadmap_section | notes |
|---|---|---|---|---|
| [[topic-slug]] | missing/shallow/partial | 8.2 | Transformers | Prerequisite for attention mechanisms |
...
```

Sort by score descending.

### Phase 5 — Rewrite gaps/study-queue.md

Overwrite `wiki/gaps/study-queue.md` with the top-N actionable items:

```markdown
---
title: Study Queue
type: study-queue
updated: <today>
---

# Study Queue

> Auto-generated by gap-agent. Edit roadmap.md or run /gap-check to refresh.

## Up Next

### 1. <Topic Name> · score: 8.2
- **Why now**: <one sentence — dependency or importance reason>
- **Suggested sources**: <2–3 specific resource types, e.g. "Attention Is All You Need paper", "Andrej Karpathy's makemore series">
- **Target page**: `wiki/concepts/<slug>.md`
- **Estimated sessions**: <1 | 2–3 | 4+>

### 2. ...
```

### Phase 6 — Log and Return

Append to `wiki/log.md`:
```
<YYYY-MM-DD HH:MM> | gap-check | wiki/gaps/list.md, wiki/gaps/study-queue.md | gap-agent
```

Return to caller:
```
{
  status: "ok",
  topics_assessed: <n>,
  gaps_found: <n>,
  gaps_closed_since_last_run: <n>,
  top_gap: "<topic>",
  queue_written: <queue_size>
}
```

## Constraints

- Only writes to `wiki/gaps/list.md` and `wiki/gaps/study-queue.md` and `wiki/log.md`. No other files.
- Never delete or archive `roadmap.md` — it is human-owned.
- Scores must be reproducible: document the rationale for each score in the `notes` column.
- If `focus` is set, only score topics within that roadmap section, but still read the full index to avoid suggesting sources the wiki already covers well.
